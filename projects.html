P1 6 Weeks Python · BeautifulSoup · CSV

Mini Google Search Engine  ·  Real Websites

GJIMT MCA × AI Integration — Semester 1 Shadow Project 1

Objective: Build a working search engine over 100 real webpages — including a web crawler, metadata database, custom ranking algorithms, and an evidence-based comparison of 5 ranking methods. AI tools are used as governed learning aids, not shortcuts.

AI Tools:  ChatGPT / Gemini  ·  GitHub Copilot (Week 4 only)  ·  Grammarly AI  ·  Notion AI

Context & Planning Manual (no AI code generation) Data & Analysis AI Augmentation
🌐 Week 1 — URL Planning: Building the Index of 100 Real Websites ▼
  • Class of 10 students divided by theme — each student finds 10 real, publicly accessible URLs from their assigned topic
  • Themes include: Education & AI, Mohali & Punjab, Technology & Coding, Careers & Business, Lifestyle & Health, Food, Sports, Travel, Science, Arts & Culture
  • Use ChatGPT / Gemini to suggest URL ideas by topic — then verify every URL manually before submission
  • Set up AI diary template using Notion AI — start logging all AI interactions from Day 1
  • Faculty reviews and approves all URLs before crawling; ensures quality and appropriateness
  • No code written yet — Week 1 is entirely planning and data curation
📄 Deliverable: url-list.txt — 100 real URLs, one per line, approved by faculty. AI diary started in Notion.
🕷️ Week 2 — Web Crawling: Collecting Data from Real Websites ▼
  • Write crawler.py — a Python script that visits all 100 URLs and extracts structured data
  • For each URL: send HTTP request → receive HTML → parse with BeautifulSoup → extract title, keywords, content (first 500 chars) → write one row to metadata.csv
  • Use time.sleep(1.5) between requests — polite crawling, avoids server blocks
  • HTML files are not saved to disk — only extracted metadata is retained; full crawl takes ~2–3 minutes
  • After crawling, open metadata.csv in Excel and manually adjust popularity scores (30–90) for each page
  • ChatGPT / Gemini permitted to explain BeautifulSoup methods and debug 403 errors — not to write the crawler
📄 Deliverable: metadata.csv — 100 rows with columns: page_id, title, url, category, keywords, content, popularity, updated, links. This is the complete search database for all remaining weeks.
🧠 Week 3 — Baseline Search Engine: Student-Written From Scratch ▼
  • Golden Rule: Every line of search_engine.py must be typed by the student — no copy-paste from AI, no "write this function for me" prompts
  • Write five functions: load_pages(), tokenise_query(), contains_ignore_case(), score_page(), print_results()
  • Scoring formula: Title match = +10 pts · Keywords match = +5 pts · Content match = +2 pts · Popularity ÷ 10 = bonus
  • Program reads metadata.csv, accepts a text query, scores all 100 pages, and displays top 5 results with real clickable URLs
  • AI permitted for explanation only: "Explain what csv.DictReader does" or "Why does sorted() need reverse=True?" — not for code generation
📄 Deliverable: search_engine.py — working search engine that reads metadata.csv, scores 100 real pages, and displays ranked results with URLs. Student can explain every single line.
🤖 Week 4 — AI-Augmented Ranking: 4 Smarter Ranking Methods ▼
  • Each student/team implements one of five ranking improvements over the baseline: Weighted Field Ranker · Popularity-Aware Ranker · Keyword Count Ranker · Freshness-Aware Ranker · Domain Trust Ranker
  • GitHub Copilot introduced this week — permitted for refactoring only: better variable names, adding docstrings, splitting long functions, removing repeated code
  • Copilot does not write ranking logic — every suggestion must be reviewed, accepted/rejected, and documented in prompts-log.md
📄 Deliverable: search_engine_v2.py — 5 ranking methods (baseline + 4 improvements). Student can explain what changed in each method and why.
📊 Week 5 — Testing & Comparison: Evidence-Based Evaluation ▼
  • Run 6 standard test queries across all 5 ranking methods: python programming · machine learning AI · punjab tourism chandigarh · data structures algorithms · career internship jobs india · cricket india sports
  • Measure for each method: Relevance (1–5) · Speed (ms via time.perf_counter) · Readability (peer-rated 1–5) · Scalability (conceptual)
  • Record all results in comparison-table.xlsx with real test data — no guesses
  • Form a written opinion: which ranking method works best and why?
📄 Deliverable: comparison-table.xlsx — side-by-side results for all 5 methods across 6 standard queries, with relevance, speed, readability, and scalability ratings.
🎓 Week 6 — Report, Viva & Live Demonstration ▼
  • final report (8–10 pages): what was built, how it works, what was found, what was learned
  • Live demo: run the search engine live, type queries, show results with real URLs in terminal
  • Viva: answer 5–8 technical questions from faculty — 5–10 minutes per student
  • complete project folder as ZIP: url-list · crawler · metadata · search engines · comparison table · AI log · report
  • Sample viva questions: "What does your crawler do?" · "How does your scoring function work?" · "What would happen with 10,000 pages?" · "Which AI tools did you use and what exactly for?"
📄 Deliverable: report.md + live demo + viva + ZIP submission of all project files.
